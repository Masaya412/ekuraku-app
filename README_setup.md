<!-- ChakraProviderをアプリ全体に適用 -->

# ChakraProvider用のファイルを作る（chakra-provider.tsxなど）

中身は以下
```tsx
"use client";

import { ChakraProvider, defaultSystem } from "@chakra-ui/react";
import { ReactNode } from "react";

type Props = {
  children: ReactNode;
};

export default function ChakraProviders({ children }: Props) {
  return <ChakraProvider value={defaultSystem}>{children}</ChakraProvider>;
}

```
# layout.tsxにChakraProviderを適用
→ layout.tsxのbodyにさっき作ったファイルの中のexportしたコンポーネントをつける

例
```tsx
import type { Metadata } from "next";
import "./globals.css";
import ChakraProviders from "./chakra-providers";

export const metadata: Metadata = {
  title: "Create Next App",
  description: "Generated by create next app",
};

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="ja">
      <body>
        <ChakraProviders>{children}</ChakraProviders>
      </body>
    </html>
  );
}
```

# Next.jsはファイルシステムルーティング

フォルダ・ファイル名がそのままルートになる
layout → childrenにpageが勝手に渡される

Node.jsは手動ルーティングなのでimportを書く必要があるが、Next.jsはapp/以下の特定のファイル名を自動認識する

ファイル名	　　 役割
page.tsx	　　　そのフォルダの URL ルートに対応する “ページ”（URLに対応したページ本体）
layout.tsx	　　そのフォルダ配下のページ全体の “レイアウト”　（親レイアウト、複数のページで共通するUIの構築をする）
loading.tsx	　　ローディング表示
error.tsx	　　　エラー表示
template.tsx	　layout のように毎回再レンダリングする型
not-found.tsx	　404 ページ
